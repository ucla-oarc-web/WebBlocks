Navbar and Navigation Children Tests

  • Nesting hierarchy and names changed to reflect WebBlocks markup requirements
  • See spreadsheet for SASS mappings and name change details
  • WebBlocks names have a prefix dash: '-' ; Original Bootstrap Classes reflect dependencies - these names are not part of the navbar scope so they have been left in original classnames for the sake of clarity -- they also show where classes need to be abstracted apart
  • Bootstrap responsive classes left in to show where needed in original context but have not been implemented since JS detection and default breakpoints will be implemented on the grid
  • An explicit .-nav class is used to cover all cases of navigation (including minor nav that cannot use the html5 nav entity -- such as footer navs
  • .-linklist is used to cover all cases where links are not part of navs (minor nav and in-page link lists) and to better manage specificity issues and (make inheritance chain more shallow).
